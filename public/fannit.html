Accessibility Audit: Non-Compliant

The Fannit website is not inclusive or accessible to people with disabilities. The scan found 53 issues, with a severity breakdown of 42 severe, 9 moderate, and 2 mild.

This raises legal risks and negatively impacts a significant portion of the user base.

Accessibility Issues: Navigation & Structure

  • The website is missing hidden skip links that allow keyboard users to bypass navigation and navigate directly to the main content or footer.
  • There are 22 instances where elements that visually function as headings are not coded with a heading tag.
  • Menu items with dropdowns are missing the aria-haspopup and aria-expanded attributes, making them difficult for screen readers to interpret.

Accessibility Issues: Elements & Readability

  • Non-button tags (e.g., <span>, <div>) used for buttons should include a role="button" attribute.
  • Links that open in a new tab are not announced to screen readers, which can cause confusion.
  • Seven instances of low text contrast were found, which can impact readability for users with low vision.
  • Some images on the site are missing alt attributes, making them inaccessible to screen readers.
